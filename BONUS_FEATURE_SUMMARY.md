## 🚀 部署步骤（超简单）

```bash
# 1. 执行数据库脚本（1分钟）
mysql -u root -p fudan_campus_info < database/analytics_enhancement.sql

# 2. 开启事件调度器（10秒）
mysql -u root -p -e "SET GLOBAL event_scheduler = ON;"

# 3. 安装前端依赖（30秒）
cd src/fudan-campus-frontend
npm install echarts

# 4. 重启服务（自动生效）
# 后端：Spring Boot会自动注册新接口
# 前端：npm run dev
```

**总耗时：不超过5分钟！**

### 可能被问到的问题
1. **为什么要用视图而不是直接在代码中查询？**
   - 答：视图提供了数据抽象层，简化了复杂查询，提高了代码复用性

2. **触发器会影响性能吗？**
   - 答：会有轻微影响，但我们只在插入时触发，且逻辑简单，影响可忽略

3. **如果数据量很大怎么办？**
   - 答：可以使用分区表、读写分离、Redis缓存等方案进一步优化

4. **为什么不直接用BI工具？**
   - 答：作为学习项目，自己实现能更好地理解底层原理，且更灵活
